Para criar mensagens de commit mais humanizadas e semânticamente adequadas criei o padrão Commit Semântico inspirado no SemVer. Trata-se de usar 3 palavras chave como prefixo da mensagem de commit:

  1. "Mudei" (MAJOR): quando fizer mudanças incompatíveis na API
  2. "Adicionei" (MINOR): quando adicionar funcionalidades mantendo compatibilidade
  3. "Alterei" (PATCH): quando corrigir falhas mantendo compatibilidade
